3.2.FINAL

Release
Version Xuggler AGPL 3.2 - "Dalton"
Date 19-August-2009
Description

"Dalton" is a performance and bug-fix release. Here are the highlights:

  • Getting and retrieving IBuffer objects from Xuggler audio, video or packet data is now much faster.
  • More memory allocation speed ups, especially in multi-threaded scenarios.
  • Xuggler now uses much less memory for IAudioSamples objects when you're just encoding data.
  • IBuffer objects now have meta-data telling you what type of bytes (e.g. int, short, long) the IBuffer thinks it's pointing to.
  • As usual, this release comes integrated with the very latest FFmpeg release.

For details, the release notes are here.

If you experience any issues, please let us know by joining the Xuggler Users group and dropping us a line.

3.0.FINAL

Release
Version 3.0.660.FINAL: "Ben"
Date 06-05-2009
Description

Xuggler 3.0 - "Ben"

Here are some of "Ben's" highlights:

  • A simpler API for decoding and encoding: MediaTool. See the new tutorials, but we think it rocks.

  • New MemoryModel technology that can speed up Xuggler programs by over 40%.

  • Improved scaling for multi-threaded programs, including the ability to interrupt blocking calls.

  • Seamless Java IO support (e.g. InputStream, and OutputStream).

  • A simpler API for getting access to video, audio and packet memory.

  • The ability to query which codecs can be encoded into which containers.

  • ... and lots more!

For details, the release notes are here.

If you experience any issues, please let us know by joining the Xuggler Users group and dropping us a line.
